## 六、经验教训

### 6.1 做得好的

| 经验 | 说明 |
|------|------|
| 代码位置重要性 | JavaScript TDZ 问题：变量引用需在定义之后 |
| 环境变量分离 | .env 用于本地开发，.env.zeabur 用于 Zeabur 部署 |
| 缓存清理 | 浏览器缓存可能导致线上版本显示旧代码 |

### 6.2 需要注意的

| 教训 | 说明 |
|------|------|
| Vite 环境变量 | VITE_ 前缀变量在构建时注入，需在构建环境设置 |
| Temporal Dead Zone | const/let 变量必须在声明后才能在同一个作用域中使用 |
