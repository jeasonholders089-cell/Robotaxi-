# 2026-04-18 AI分析预估数量修复 + Zeabur环境变量配置

## 会话时间
2026-04-18

## 背景
用户反馈：在反馈管理页面，按评分 1-4 筛选后，反馈列表显示"共 120 条反馈"，但 AI 智能分析区域仍显示"预估 1500 条数据"。

## 问题分析

### 问题1：AI分析预估数量显示错误
**现象**：筛选条件（评分 1-4）下，反馈列表显示 120 条，但 AI 智能分析显示预估 1500 条。

**根本原因**：
- 代码中 `estimatedCount` 使用的是 `stats?.total_count`（来自 stats API，返回的是未筛选的全量数据）
- 正确应该使用 `data?.total`（来自 feedbacks API，返回筛选后的数据）

**修复方案**：
```javascript
// 修改前
const estimatedCount = stats?.total_count?.toLocaleString() ?? '0'

// 修改后
const estimatedCount = data?.total?.toLocaleString() ?? '0'
```

### 问题2：JavaScript 初始化错误
**现象**：浏览器控制台报错 `ReferenceError: Cannot access 'R' before initialization`

**根本原因**：
- `estimatedCount` 在第 125 行使用 `data?.total`
- 但 `data` 是通过 `useFeedbacks()` hook 在第 136 行才定义的
- JavaScript temporal dead zone 导致初始化错误

**修复方案**：
- 将 `estimatedCount` 定义移到 `useFeedbacks()` 调用之后

### 问题3：Zeabur 部署环境变量问题
**现象**：线上版本（https://groclawa.com/feedbacks）控制台报错 `localhost:8080 Failed to load resource: net::ERR_CONNECTION_REFUSED`

**根本原因**：
- `client/.env` 文件中 `VITE_API_BASE_URL=http://localhost:8080`
- 部署到 Zeabur 后没有正确的 API 地址

**修复方案**：
1. 在 Zeabur 控制台为前端服务设置环境变量：`VITE_API_BASE_URL=https://api.groclawa.com`
2. 创建 `client/.env.zeabur` 文件
3. 更新 `client/zeabur.yaml` 添加环境变量配置
