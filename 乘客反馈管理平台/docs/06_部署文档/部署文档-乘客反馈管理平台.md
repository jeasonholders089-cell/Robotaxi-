# 部署文档: Robotaxi 乘客反馈管理平台

## 2. 环境准备

### 2.1 阿里云服务器初始化

1. 购买并启动阿里云 ECS 实例
2. 选择操作系统：Ubuntu 20.04 LTS
3. 配置安全组规则：
   - 开放 22 端口（SSH）
   - 开放 80 端口（HTTP）
   - 开放 443 端口（HTTPS）
   - 开放 8000 端口（后端 API）

### 2.2 安装宝塔面板

```bash
# Ubuntu 20.04 安装宝塔面板
wget -O install.sh https://download.bt.cn/install/install-ubuntu_6.0.sh && sudo bash install.sh

# 安装完成后，记录面板地址和登录信息
```

### 2.3 宝塔面板安装软件

通过宝塔面板应用商店安装：

| 软件 | 版本 | 用途 |
|------|------|------|
| Nginx | 最新版 | Web 服务器 |
| MySQL | 8.0+ | 数据库 |
| Python | 3.10+ | 后端运行时 |

## 3. 项目部署

### 3.1 创建数据库

1. 登录宝塔面板 → 数据库
2. 创建数据库：
   - 数据库名：`robotaxi`
   - 用户名：`robotaxi`
   - 密码：[设置强密码]
   - 编码：`utf8mb4`

### 3.2 导入数据库表结构

在宝塔面板 phpMyAdmin 中执行以下 SQL：

```sql
-- 创建数据库（如果尚未创建）
CREATE DATABASE IF NOT EXISTS robotaxi DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
USE robotaxi;

-- 反馈主表
CREATE TABLE `feedback` (
  `id` VARCHAR(20) NOT NULL COMMENT '反馈ID，主键，格式：FB+日期+序号',
  `trip_id` VARCHAR(20) NOT NULL COMMENT '行程ID',
  `passenger_id` VARCHAR(20) NOT NULL COMMENT '乘客ID（脱敏）',
  `vehicle_id` VARCHAR(30) NOT NULL COMMENT '车辆ID，格式：Robotaxi-{城市}-{序号}',
  `city` VARCHAR(20) NOT NULL COMMENT '城市',
  `route` VARCHAR(100) NOT NULL COMMENT '路线名称',
  `route_start` VARCHAR(50) NOT NULL COMMENT '起点',
  `route_end` VARCHAR(50) NOT NULL COMMENT '终点',
  `trip_time` DATETIME NOT NULL COMMENT '行程开始时间',
  `trip_duration` INT NOT NULL COMMENT '行程时长（分钟）',
  `rating` TINYINT NOT NULL COMMENT '评分 1-5',
  `feedback_text` TEXT NOT NULL COMMENT '乘客原始反馈',
  `feedback_type` JSON DEFAULT NULL COMMENT 'AI分类结果，数组格式',
  `sentiment` VARCHAR(10) DEFAULT NULL COMMENT '情感倾向：正面/中性/负面',
  `keywords` JSON DEFAULT NULL COMMENT 'AI提取的关键词',
  `status` VARCHAR(20) NOT NULL DEFAULT 'pending' COMMENT '跟进状态',
  `handler` VARCHAR(50) DEFAULT NULL COMMENT '处理人',
  `handler_notes` TEXT DEFAULT NULL COMMENT '处理记录',
  `handled_at` DATETIME DEFAULT NULL COMMENT '最近处理时间',
  `ai_summary` TEXT DEFAULT NULL COMMENT '该反馈的AI摘要',
  `feedback_channel` VARCHAR(16) DEFAULT 'App' COMMENT '反馈渠道：App/小程序/电话/Web',
  `reply_text` TEXT DEFAULT NULL COMMENT '客服回复内容',
  `reply_time` DATETIME DEFAULT NULL COMMENT '客服回复时间',
  `created_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '记录创建时间',
  `updated_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '最后修改时间',
  PRIMARY KEY (`id`),
  INDEX `idx_trip_time` (`trip_time`),
  INDEX `idx_rating` (`rating`),
  INDEX `idx_city` (`city`),
  INDEX `idx_status` (`status`),
  INDEX `idx_created_at` (`created_at`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='反馈主表';

-- 路线预设表
CREATE TABLE `route_preset` (
  `id` INT NOT NULL AUTO_INCREMENT COMMENT '主键',
  `city` VARCHAR(20) NOT NULL COMMENT '城市',
  `route_name` VARCHAR(100) NOT NULL COMMENT '路线名称',
  `route_start` VARCHAR(50) NOT NULL COMMENT '起点',
  `route_end` VARCHAR(50) NOT NULL COMMENT '终点',
  `active` TINYINT NOT NULL DEFAULT 1 COMMENT '是否启用',
  `created_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_route` (`city`, `route_start`, `route_end`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='路线预设表';
```

### 3.4 部署后端

1. 在宝塔面板创建 Python 项目
2. 配置项目路径：`/www/wwwroot/robotaxi-feedback/server`
3. 安装依赖：

```bash
cd /www/wwwroot/robotaxi-feedback/server
pip install -r requirements.txt
```

4. 配置环境变量（通过宝塔面板或创建 .env 文件）：

```bash
# /www/wwwroot/robotaxi-feedback/server/.env
DATABASE_URL=mysql+pymysql://robotaxi:你的密码@127.0.0.1:3306/robotaxi
KIMI_API_KEY=你的Kimi_API密钥
MINIMAX_API_KEY=你的MiniMax_API密钥
DEBUG=false
LOG_LEVEL=info
CORS_ORIGINS=https://你的域名
```

5. 启动后端服务（使用 Gunicorn）：

```bash
cd /www/wwwroot/robotaxi-feedback/server
gunicorn -w 4 -b 127.0.0.1:8000 app.main:app --daemon
```

### 3.5 部署前端

1. 本地构建前端：

```bash
cd client
npm install
npm run build
```

2. 将 `dist` 目录上传到服务器：`/www/wwwroot/robotaxi-feedback/client`

3. 配置 Nginx：

```nginx
# /etc/nginx/conf.d/robotaxi.conf
server {
    listen 80;
    server_name your-domain.com;  # 替换为你的域名

    root /www/wwwroot/robotaxi-feedback/client/dist;
    index index.html;

    # 前端路由支持
    location / {
        try_files $uri $uri/ /index.html;
    }

    # API 反向代理
    location /api {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
}
```

4. 重启 Nginx：

```bash
nginx -t && nginx reload
```

## 4. 环境变量配置

### 4.1 后端环境变量

| 变量名 | 必填 | 说明 | 示例 |
|--------|------|------|------|
| DATABASE_URL | 是 | MySQL 连接字符串 | `mysql+pymysql://user:pass@host:3306/db` |
| KIMI_API_KEY | 是 | Kimi API 密钥 | `moonshot-xxx` |
| MINIMAX_API_KEY | 是 | MiniMax API 密钥 | `eyJxxx` |
| DEBUG | 否 | 调试模式 | `false` |
| LOG_LEVEL | 否 | 日志级别 | `info` |
| CORS_ORIGINS | 否 | 允许的跨域源 | `https://your-domain.com` |

### 4.2 前端环境变量

| 变量名 | 必填 | 说明 | 示例 |
|--------|------|------|------|
| VITE_API_BASE_URL | 是 | API 基础地址 | `https://api.your-domain.com` |

## 6. 运维维护

```bash

journalctl -u robotaxi-backend -f
```

## 7. 故障排查

### 7.1 常见问题

| 问题 | 可能原因 | 解决方案 |
|------|----------|----------|
| 页面 502 | 后端服务未启动 | 检查 gunicorn 进程 |
| API 请求失败 | 跨域配置问题 | 检查 CORS 配置 |
| 数据库连接失败 | 密码错误或权限不足 | 检查 DATABASE_URL |
| AI 功能不可用 | API 密钥错误 | 检查环境变量中的密钥 |

### 7.2 检查命令

```bash
# 检查进程
ps aux | grep gunicorn
ps aux | grep nginx

# 检查端口占用
netstat -tlnp | grep 8000
netstat -tlnp | grep 80

# 检查日志
tail -100 /var/log/nginx/error.log
```
