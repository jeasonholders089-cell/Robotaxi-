
AI PM Coding 能力测试
Robotaxi 乘客反馈管理平台

滴滴自动驾驶  |  AI 产品经理岗位
完成时限：48小时  |  工具要求：Claude Code
 
一、测试概述
背景与目标
滴滴自动驾驶 Robotaxi 服务未来会在多个城市开放运营，随着订单规模增长，乘客反馈数据快速积累。当前团队需要一个内部工具来高效管理和分析这些反馈，以数据驱动产品体验的持续优化。
请你使用 Claude Code，从 0 到 1 构建一个「Robotaxi 乘客反馈管理平台」，展现你用 AI 工具独立交付产品的能力。


二、功能要求
模块一：反馈列表与管理
构建一个可用的反馈列表页面，支持运营团队日常查看和管理乘客反馈。
1.	反馈列表展示：支持分页、排序（按时间、评分）
2.	多维度筛选：按评分范围、时间范围、路线、反馈类型等条件筛选
3.	反馈详情：点击可查看单条反馈的完整信息
模块二：数据统计仪表盘
构建数据可视化仪表盘，帮助团队快速了解反馈整体情况。
1.	核心指标卡片：总反馈数、平均评分、好评率/差评率等
2.	趋势图表：反馈数量及评分的时间趋势
3.	分布分析：按路线、时间段、反馈类型等维度的分布
模块三：AI 智能分析
集成大模型 API，为反馈数据提供智能化分析能力。
1.	自动分类：对乘客反馈文本进行自动分类（如：行驶体验、车内环境、接驾体验、路线规划、安全感受等）
2.	摘要生成：对批量反馈生成智能摘要，提炼核心问题和用户说词
3.	产品建议：基于反馈分析结果，自动生成产品优化建议

三、数据要求
请自行构造模拟数据。数据的设计质量本身也是评估的一部分——好的模拟数据能体现你对业务场景的理解。
参考字段（可自行扩展）
以下字段为参考建议，你可以根据产品设计需要自由调整、增减字段：
参考字段	说明
反馈 ID / 乘客 ID / 行程 ID	基本标识信息
评分	如 1-5 分
反馈文本	乘客的文字评价
路线 / 城市	地理信息
行程时间 / 时长	时间信息
车辆 ID	车辆标识
其他	欢迎根据你的产品设计添加更多有价值的字段

基本要求
•	数据量不少于 100 条
•	反馈文本应贴近真实乘车场景，覆盖正面、负面、中性评价
•	可以使用 AI 生成模拟数据
