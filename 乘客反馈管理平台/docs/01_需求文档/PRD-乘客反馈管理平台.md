# PRD: Robotaxi 乘客反馈管理平台

## 1. 产品概述

### 1.1 产品背景

随着滴滴自动驾驶 Robotaxi 服务在多个城市的逐步开放运营，订单规模持续增长，乘客反馈数据快速积累。目前反馈数据分散在多个渠道，缺乏统一的结构化管理平台，导致运营团队难以快速获取有价值的洞察，产品迭代缺乏数据支撑。

### 1.2 产品定位

Robotaxi 乘客反馈管理平台是面向滴滴自动驾驶运营团队和产品的内部工具，集成反馈数据的采集、管理、统计分析和 AI 智能分析功能，帮助团队高效处理乘客反馈，挖掘产品优化机会，提升乘客出行体验。

### 1.3 产品目标

- 建立统一的反馈数据管理入口，实现反馈全生命周期管理
- 提供直观的数据统计仪表盘，快速掌握服务运行状态
- 借助 AI 能力实现反馈智能分类、摘要生成和产品建议，提升分析效率

### 1.4 核心价值

| 角色 | 价值点 |
|------|--------|
| 运营团队 | 快速查询和处理乘客反馈，提升响应效率 |
| 产品经理 | 获得数据驱动的产品优化建议，加速迭代 |
| 数据分析师 | 获取结构化的反馈数据和分析报表 |
| 管理决策者 | 掌握服务整体运行状态和用户满意度趋势 |

## 3. 功能详情

### 3.0 模块零：工作台（Landing Page）

#### 3.0.1 功能描述

工作台是用户登录后的默认首页，提供系统关键数据概览和快捷操作入口，帮助用户第一时间了解当日核心数据和待处理任务。

#### 3.0.2 功能点

| 功能 | 说明 |
|------|------|
| 今日概览 | 展示今日反馈总数、待处理数、好评率等关键指标 |
| 待办提醒 | 展示需要紧急处理的反馈数量和类型 |
| 趋势预览 | 显示近7天反馈数量趋势简图 |
| AI 摘要 | 显示 AI 生成的一句话总结 |
| 快捷入口 | 跳转至各功能模块的快捷按钮 |

#### 3.0.3 交互说明

- 概览指标卡片点击可跳转至对应筛选结果

- 快捷入口点击直接跳转至对应页面

### 3.1 模块一：反馈列表与管理

#### 3.1.1 反馈列表展示

**功能描述**

反馈列表是系统的核心入口，展示所有乘客提交的反馈记录，支持分页浏览和排序操作。

**页面布局**

- 顶部：筛选条件区域（折叠/展开）
- 左侧/顶部：筛选标签展示区
- 主区域：反馈数据表格
- 底部：分页控件

**表格字段**

| 字段 | 说明 | 样式 |
|------|------|------|
| 反馈ID | 系统生成的唯一标识 | 文本，蓝色链接可点击 |
| 评分 | 乘客评分（1-5星） | 图标+数字 |
| 反馈类型 | 自动分类或手动标注 | 彩色标签 |
| 反馈摘要 | 文本前100字符 | 文本，超长省略 |
| 路线 | 起点-终点 | 文本 |
| 城市 | 行程所在城市 | 标签 |
| 行程时间 | 乘客下单时间 | yyyy-mm-dd HH:mm |
| 状态 | 处理状态 | 待处理/处理中/已解决/已关闭 |

**分页规则**

- 默认每页20条记录
- 支持10/20/50/100条每页切换
- 显示总记录数和当前页范围
- 支持跳转到指定页

**排序规则**

- 默认排序：按行程时间倒序（最新在前）
- 可排序列：时间（升序/降序）、评分（升序/降序）
- 显示当前排序状态和排序方向箭头

#### 3.1.2 多维度筛选

**筛选条件**

| 筛选维度 | 筛选项 | 操作方式 |
|----------|--------|----------|
| 评分范围 | 1星、2星、3星、4星、5星 | 多选checkbox |
| 时间范围 | 开始日期、结束日期 | 日期选择器 |
| 路线 | 支持搜索的 dropdown | 模糊搜索+多选 |
| 反馈类型 | 行驶体验/车内环境/接驾体验/路线规划/安全感受/其他 | 多选checkbox |
| 城市 | 支持搜索的 dropdown | 模糊搜索+多选 |
| 状态 | 待处理/处理中/已解决/已关闭 | 多选checkbox |
| 关键词 | 反馈文本关键词 | 文本输入框 |

**交互行为**

- 筛选条件区域默认折叠，点击"展开筛选"按钮显示完整筛选面板
- 选择筛选条件后，实时更新标签展示区的已选条件标签
- 每个已选标签可单独移除，点击"清除全部"一键移除所有筛选
- 筛选条件变更后自动触发列表刷新，无需手动点击查询按钮
- 筛选条件支持保存为常用筛选项，方便快速调用

#### 3.1.4 批量操作

**功能描述**

支持对多条反馈进行批量处理，提升运营效率。

**批量操作类型**

| 操作 | 说明 | 适用场景 |
|------|------|----------|
| 批量更新状态 | 批量修改处理状态 | 集中处理同类反馈 |
| 批量标记 | 添加标签/分类标记 | 数据标注 |
| 批量导出 | 导出选中反馈数据 | 离线分析 |
| 批量删除 | 删除选中的反馈 | 清理脏数据（需二次确认） |

**交互流程**

1. 用户在列表勾选多条反馈（支持全选当前页/全选全部）
2. 底部弹出批量操作栏，显示已选数量
3. 点击批量操作按钮，弹出确认对话框
4. 确认后执行操作，显示操作结果

**批量导出规则**

- 支持导出格式：Excel、CSV
- 导出字段可配置
- 单次最多导出 1000 条
- 超出提示分批导出

### 3.2 模块二：数据统计仪表盘

#### 3.2.1 核心指标卡片

**指标卡片设计**

页面顶部展示4-6个核心指标卡片，采用卡片式布局，每个卡片包含：

| 指标名称 | 计算方式 | 展示形式 |
|----------|----------|----------|
| 总反馈数 | 统计周期内反馈记录总数 | 数字 + 较上周变化百分比（红↓/绿↑） |
| 平均评分 | 反馈评分总和 / 反馈数量 | 数字（1位小数）+ 星级图标 |
| 好评率 | 评分>=4的反馈占比 | 百分比 + 趋势箭头 |
| 差评率 | 评分<=2的反馈占比 | 百分比 + 趋势箭头 |
| 待处理反馈 | 状态为待处理的反馈数量 | 数字 + 橙色警示 |
| 处理完成率 | 已解决+已关闭 / 总反馈数 | 百分比 + 进度条 |

**交互行为**

- 指标卡片支持点击跳转到对应的筛选列表（如点击"差评率"跳转至评分1-2星的筛选结果）
- 卡片数据随顶部时间范围筛选器联动

#### 3.2.2 趋势图表

**反馈数量趋势图**

- 图表类型：折线图/面积图
- X轴：时间（按天/周/月）
- Y轴：反馈数量
- 支持多条曲线：总反馈、好评、差评
- 交互：hover显示详细数值，支持缩放和区间选择

**平均评分趋势图**

- 图表类型：折线图
- X轴：时间（按天/周/月）
- Y轴：平均评分（1-5）
- 添加评分标准线（4.0为基准线）
- 交互：hover显示详细数值

**图表控件**

- 时间粒度切换：日/周/月
- 时间范围选择：近7天/近30天/近90天/自定义
- 导出按钮：导出图表为PNG/PDF

#### 3.2.3 分布分析

**评分分布图**

- 图表类型：柱状图/饼图
- 展示各评分（1-5星）的反馈数量和占比
- 点击柱子可跳转至对应评分范围的反馈列表

**反馈类型分布图**

- 图表类型：饼图/环形图
- 展示各类别的反馈数量和占比
- 支持切换为柱状图展示

**城市分布图**

- 图表类型：地图热力图 + 柱状图
- 地图展示各城市反馈数量热力分布
- 柱状图展示Top10城市排名

**路线分布图**

- 图表类型：柱状图
- 展示反馈数量Top10的路线
- 点击柱子跳转至该路线的反馈列表

**时间段分布图**

- 图表类型：柱状图
- X轴：时间段（凌晨/上午/中午/下午/晚上）
- 展示各时间段的反馈数量分布

### 3.3 模块三：AI 智能分析

#### 3.3.1 自动分类

**分类体系**

| 一级分类 | 二级分类 | 说明 |
|----------|----------|------|
| 行驶体验 | 行驶平稳性、加减速体验、车道保持、转弯体验 | 与驾驶行为相关 |
| 车内环境 | 温度舒适度、空间舒适度、噪音水平、清洁度 | 与车内环境相关 |
| 接驾体验 | 等候时长、上车点准确性、司机响应 | 与预约和接驾相关 |
| 路线规划 | 路线合理性、到达效率、导航准确性 | 与导航和路线相关 |
| 安全感受 | 紧急情况处理、安全配置、应急响应 | 与安全感相关 |
| 服务态度 | 客服响应、问题解决效率、沟通体验 | 与服务相关 |
| 其他 | 不属于以上类别 | 兜底分类 |

**实现方式**

- 调用 Kimi API 或 MiniMax API 进行文本分类
- 对新提交的反馈自动进行分类，分类结果存储到数据库
- 支持人工校正分类结果，校正后更新AI模型参考

**交互功能**

- 反馈列表中显示分类标签
- 详情页显示完整分类结果和置信度
- 支持按分类维度筛选反馈

#### 3.3.2 摘要生成

**批量摘要功能**

- 选择多条反馈（可按筛选条件批量选择）
- 调用 AI API 生成汇总摘要
- 摘要内容包括：整体满意度、主要好评点、主要投诉点、改进建议

**单条摘要功能**

- 对单条反馈生成简明摘要（50-100字）
- 自动提取关键词（3-5个）

**摘要展示位置**

- 反馈列表中显示摘要预览
- 详情页显示完整摘要和关键词

#### 3.3.3 产品建议

**分析维度**

- 识别高频投诉问题和负面关键词
- 分析评分变化趋势
- 对比不同城市/路线的反馈差异

**建议生成**

- 基于分析结果自动生成产品优化建议
- 建议分级：紧急/重要/一般
- 建议格式：问题描述 + 改进建议 + 预期效果

**产品建议看板**

- 独立页面展示待处理的产品建议
- 支持建议状态管理（待确认/已采纳/已上线/已驳回）
- 支持导出产品建议报告
