## 1. 概述

### 1.1 背景与目的

当前仪表盘 UI 存在典型「AI 生成风格」问题，缺乏品牌个性与视觉差异化。本迭代基于 **impeccable** 设计规范进行品质升级，保持滴滴橙色品牌主题，打造更专业、更有辨识度的数据仪表盘。

### 1.2 设计原则

| 原则 | 说明 |
|------|------|
| **品牌一致性** | 橙色 (#FF6033) 作为主色调贯穿所有图表和组件 |
| **克制用色** | 配色以滴滴橙为主，避免彩虹式均匀配色 |
| **层次分明** | 通过尺寸、权重、间距建立清晰信息层级 |
| **去模板化** | 打破「卡片+图表」的模板化布局，建立视觉节奏 |

### 1.3 术语定义

| 术语 | 定义 |
|------|------|
| OKLCH | 感知均匀的色彩空间，比 HSL 更适合做配色系统 |
| 设计 Token | 设计系统中定义的变量（如颜色、间距、字体） |
| AI Slop | 缺乏个性的通用 AI 生成设计模式 |

## 3. 解决方案

### 3.1 概览指标卡片优化

#### P1 & P2: 网格与层级优化

**方案**: 采用 4 列为主网格，重要指标跨列

```
变更前: grid-cols-2 md:grid-cols-3 lg:grid-cols-6 (6等列)
变更后: grid-cols-2 lg:grid-cols-4 (4列，主指标跨2列)
```

**布局示意**:
```
┌──────────────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐
│    总反馈数       │ │ 平均评分  │ │  好评率   │ │  差评率   │
│    (跨1列)       │ │          │ │          │ │          │
├──────────────────┤ ├──────────┤ ├──────────┤ ├──────────┤
│                  │ │          │ │          │ │          │
│                  │ │          │ │          │ │          │
└──────────────────┘ └──────────┘ └──────────┘ ┌──────────┐
                                               │  待处理   │
┌──────────────────┐ ┌──────────┐             │  (跨2列)  │
│  处理完成率(跨2列) │ │          │             │           │
└──────────────────┘ └──────────┘             └───────────┘
```

#### P3: 趋势箭头语义优化

| 场景 | 指标 | 向上箭头含义 | 颜色 |
|------|------|-------------|------|
| 总反馈数 | 数量增加 | 好（业务增长） | success green |
| 平均评分 | 评分增加 | 好（满意度提升） | success green |
| 好评率 | 比率增加 | 好（满意度提升） | success green |
| 差评率 | 比率增加 | 坏（问题增多） | error red |
| 待处理 | 数量增加 | 坏（积压增加） | error red |

### 3.2 趋势图表优化

#### P4: 移除渐变面积填充

**方案 A**: 纯折线（推荐，符合滴滴简洁风格）
```typescript
series: [{
  type: 'line',
  smooth: false,           // 移除平滑
  lineStyle: {
    color: '#FF6033',
    width: 2,
  },
  // 移除 areaStyle
}]
```

**方案 B**: 细柱状图（适合展示离散数据）
```typescript
series: [{
  type: 'bar',
  barWidth: '40%',
  itemStyle: {
    color: '#FF6033',
    borderRadius: [2, 2, 0, 0],
  },
}]
```

#### P6: 基准线颜色调整

```typescript
markLine: {
  lineStyle: {
    color: '#CCCCCC',  // 中性灰，不抢主色调风头
    type: 'dashed',
  },
  data: [{ yAxis: 4.0 }],
  label: {
    color: '#999999',
  },
}
```

### 3.3 分布图表优化

#### P7: 饼图替代方案

**评分分布** → 使用「星星进度条」替代饼图
```
★★★★★ ████████████████ 78%
★★★★   ██████            15%
★★★     ███               5%
```

**类型分布** → 使用「水平进度条」替代饼图
```
行驶体验   ████████████████████ 45%
车内环境   █████████           25%
接驾体验   ██████              18%
路线规划   ███                 8%
安全感受   █                   4%
```

#### P8: 配色系统重构

**原则**: 以滴滴橙为主色，通过透明度建立层次

```typescript
// Didi OKLCH 配色系统
const colors = {
  primary: '#FF6033',           // 滴滴橙

  // 橙色调渐变（用于同一数据系列的不同状态）
  orangeLight: 'oklch(85% 0.08 40)',   // 浅橙
  orangeMid: 'oklch(70% 0.12 40)',     // 中橙
  orangeDark: 'oklch(55% 0.15 40)',    // 深橙

  // 中性色（带微橙色偏）
  neutral: {
    900: 'oklch(20% 0.02 40)',   // 深色文字
    700: 'oklch(40% 0.02 40)',   // 正文
    500: 'oklch(60% 0.02 40)',   // 次要文字
    300: 'oklch(80% 0.02 40)',   // 边框
    100: 'oklch(95% 0.01 40)',   // 背景
  },

  // 功能色（保持高饱和度以确保可辨识）
  success: '#52C41A',
  warning: '#FAAD14',
  error: '#F5222D',
}
```

**图表配色应用**:
| 图表 | 配色方案 |
|------|----------|
| 评分分布 | 5星=success, 4星=#95DE64, 3星=warning, 2星/1星=error |
| 类型分布 | 统一使用 primary 橙色系，由深到浅区分 |
| 城市分布 | primary 单色 |
| 时段分布 | primary 单色热力渐变 |

### 3.4 布局优化

**变更原则**:
- 指标卡片区域：保留卡片
- 趋势图表区域：移除卡片包装，使用底部边框线分隔
- 分布图表区域：交替使用卡片/无卡片，打破节奏

**新布局示意**:

#### P12: 标题对比度提升

```typescript
// 变更前
<h3 className="text-sm font-medium text-gray-700 mb-4">

// 变更后
<h3 className="text-sm font-medium" style={{ color: '#333333' }}>
```

#### P14: 非对称布局探索

**方案**: 第一层趋势图使用 2:1 比例，打破等分布局
```
变更前: lg:grid-cols-2 (各50%)
变更后: lg:grid-cols-3 (趋势图占2份，评分趋势占1份)
```

## 5. 组件变更清单

### 5.1 StatCard 组件

| 变更项 | 变更内容 |
|--------|----------|
| 尺寸 | 支持 `size: 'sm' \| 'md' \| 'lg'` |
| 跨度 | 支持 `span: 1 \| 2` 控制网格跨越 |
| 趋势颜色 | 根据指标语义自动配色 |

### 5.2 TrendChart 组件

| 变更项 | 变更内容 |
|--------|----------|
| 样式类型 | 支持 `style: 'line' \| 'bar'` |
| 渐变填充 | 默认关闭，可通过 `showArea: true` 开启 |
| 平滑曲线 | 默认关闭，可通过 `smooth: true` 开启 |
| 基准线 | 颜色改为中性灰 |

### 5.3 DistributionChart 组件

| 变更项 | 变更内容 |
|--------|----------|
| 饼图 | 可选展示方式：环形/实心/进度条 |
| 配色 | 支持传入语义化配色数组 |
| 图例 | 支持位置：`right` \| `bottom` \| `none` |

### 5.4 Dashboard 页面

| 变更项 | 变更内容 |
|--------|----------|
| 网格布局 | 从 6 列改为 4 列基础网格 |
| 卡片策略 | 趋势图区域移除卡片包装 |

## 6. 验收标准

### 6.1 设计品质检查

- [ ] 无渐变面积填充（除非明确业务需求）
- [ ] 配色以滴滴橙为主色，功能色保持高饱和度
- [ ] 标题文字对比度 ≥ 4.5:1
- [ ] 指标卡片有清晰的视觉层级
- [ ] 图表间布局有节奏感，非均匀分布

### 6.2 功能检查

- [ ] 指标卡片点击跳转正常
- [ ] 时间范围切换正常
- [ ] 图表 Tooltip 显示正常
- [ ] 响应式布局正常

### 6.3 AI Slop 自检清单

> 如果展示给设计师说「这是 AI 生成的」，应该感到羞愧的项目特征：

- [ ] 无渐变色块背景
- [ ] 无彩虹渐变配色
- [ ] 无「card内嵌chart」的标准化模板
- [ ] 无圆润过度的曲线
- [ ] 无「紫色系按钮 + 青色数据可视化」的组合
