# v1.3 需求迭代文档 - AI分析功能优化

**版本**: v1.3
**日期**: 2026-04-17
**状态**: 已完成

---

## 1. 概述

### 1.1 背景

v1.2 版本实现了筛选条件联动，但 AI 分析功能存在以下问题：

1. **筛选参数未传递**：选择"当前筛选"时，前端未将全局筛选条件传递给后端 API
2. **后端 API 筛选能力不足**：AI 分析接口仅支持 `start_date`、`end_date`、`city` 三个筛选参数，缺少 `rating_min/max`、`status`、`feedback_type`、`keyword` 等支持
3. **问题分类饼图数据错误**：饼图错误地使用 AI 生成的建议数据，而非实际的问题分类分布

### 1.2 目标

1. 实现 AI 分析与全局筛选条件的完整联动
2. 扩展后端 AI API 的筛选参数支持
3. 修复问题分类饼图的数据来源

---

## 2. 功能需求

### 2.1 前端筛选联动

**当前问题**：选择"当前筛选"时，`analysisScope === 'current'` 分支传递 `undefined` 作为所有筛选参数

**修复方案**：
- 使用 `useFilterStore` 获取全局筛选条件
- 当 `analysisScope === 'current'` 时，传递以下参数：
  - `start_date` / `end_date`
  - `city`
  - `rating_min` / `rating_max`
  - `status[]`
  - `feedback_type[]`
  - `keyword`

### 2.2 后端 AI API 扩展

#### 2.2.1 AI 摘要接口 (`POST /api/ai/summary`)

新增筛选参数：
- `rating_min`: 最低评分筛选
- `rating_max`: 最高评分筛选
- `status[]`: 状态多选筛选
- `feedback_type[]`: 反馈类型多选筛选
- `keyword`: 关键词筛选

#### 2.2.2 AI 建议接口 (`POST /api/ai/suggestions`)

新增筛选参数：
- `rating_min`: 最低评分筛选
- `rating_max`: 最高评分筛选
- `status[]`: 状态多选筛选
- `feedback_type[]`: 反馈类型多选筛选
- `keyword`: 关键词筛选

### 2.3 问题分类饼图修复

**当前问题**：饼图使用 `suggestions?.suggestions`（AI 建议数据）进行渲染

**修复方案**：
- 在 `AISuggestionsResponse` 中新增 `type_distribution` 字段
- 前端饼图使用 `suggestions?.type_distribution` 进行渲染
