# 遇到的问题

## 问题1：前端 API 请求 404

**原因**：前端 `client.ts` 的 `baseURL` 已设为 `/api`，但 API 路径又重复加了 `/api` 前缀，导致请求变成 `api/api/feedbacks`。

**修复**：
- `feedback.ts`、`stats.ts`、`ai.ts` 中移除重复的 `/api` 前缀

---

## 问题2：状态字段前后端不一致

**原因**：
- 数据库中混合存储了中英文状态值
- 前端期望英文（`pending`/`processing`/`resolved`/`closed`）
- 后端实际返回中文（`待处理`/`处理中`/`已解决`）

**修复**：
1. 执行数据迁移脚本 `migrate_status.py`，将数据库中的中文状态值统一改为英文
2. 前端恢复为只支持英文状态值

**迁移结果**：
- `待处理` → `pending`: 54 条
- `处理中` → `processing`: 51 条
- `已解决` → `resolved`: 33 条
- `已关闭` → `closed`: 12 条

---

## 设计规范

状态值统一使用**英文枚举**：
- `pending` - 待处理
- `processing` - 处理中
- `resolved` - 已解决
- `closed` - 已关闭

UI 显示时通过 `statusConfig` 映射到对应的中文 `label`。
