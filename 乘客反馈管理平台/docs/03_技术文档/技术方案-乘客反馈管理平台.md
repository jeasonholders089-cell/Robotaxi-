## 1. 技术架构概述

### 1.1 整体架构

本平台采用前后端分离架构，前端基于 React + TypeScript 构建单页应用，后端基于 Python + FastAPI 提供 RESTful API 服务，数据库使用 MySQL，AI 能力集成 Kimi API 和 MiniMax API。

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                              前端 (React + TypeScript)                        │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐     │
│  │  反馈列表页   │  │  仪表盘页面   │  │  AI 分析页   │  │  反馈详情    │     │
│  └──────┬───────┘  └──────┬───────┘  └──────┬───────┘  └──────┬───────┘     │
│         └─────────────────┴─────────────────┴─────────────────┘             │
│                                    │                                        │
│                          React Query / Axios                                │
└────────────────────────────────────┼────────────────────────────────────────┘
                                     │ HTTP REST API
┌────────────────────────────────────┼────────────────────────────────────────┐
│                              后端 (Python + FastAPI)                         │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐     │
│  │  反馈管理 API │  │  统计分析 API │  │  AI 分析 API │  │  中间件      │     │
│  └──────┬───────┘  └──────┬───────┘  └──────┬───────┘  └──────────────┘     │
│         └─────────────────┴─────────────────┘                               │
│                                    │                                        │
│                           MySQL 数据库                                       │
└────────────────────────────────────┴────────────────────────────────────────┘
                                     │
                    ┌────────────────┴────────────────┐
                    │         外部 AI 服务            │
                    │   Kimi API  /  MiniMax API     │
                    └─────────────────────────────────┘
```

## 2. 前端技术方案

### 2.1 技术选型

| 技术 | 选型理由 |
|------|----------|
| React 18 | 函数式组件 + Hooks，提供更好的性能和并发特性 |
| TypeScript | 类型安全，减少运行时错误，提升代码可维护性 |
| Tailwind CSS | 原子化 CSS，快速构建一致性的 UI，减少 CSS 文件体积 |
| Vite | 开发体验好，热更新快，构建速度快 |
| Zustand | 状态管理轻量简单，相比 Redux 更容易上手 |
| TanStack Query | 数据获取、缓存和更新逻辑封装，减少样板代码 |
| ECharts | 图表能力丰富，支持大数据量渲染，社区活跃 |

### 2.2 项目结构

前端项目采用 Feature-based 组织方式，按功能模块划分代码：

```
client/src/
├── api/                 # API 层：封装所有 HTTP 请求
├── components/          # 公共组件库
│   ├── ui/             # 基础 UI 组件（Button、Input、Modal 等）
│   └── *.tsx          # 业务组件
├── pages/              # 页面组件（路由级别组件）
├── stores/             # Zustand 全局状态
├── hooks/              # 自定义 Hooks（复用逻辑）
├── types/              # TypeScript 类型定义
├── utils/              # 工具函数
└── styles/             # 全局样式
```

### 2.3 核心组件设计

#### 2.3.1 FeedbackTable（反馈列表表格）

**职责**：展示反馈数据列表，支持分页、排序、行选择

**Props 接口**：
```typescript
interface FeedbackTableProps {
  data: Feedback[];
  loading: boolean;
  pagination: {
    page: number;
    pageSize: number;
    total: number;
  };
  onPageChange: (page: number) => void;
  onSort: (sortBy: 'time' | 'rating', order: 'asc' | 'desc') => void;
  onRowClick: (feedback: Feedback) => void;
  selectedIds: string[];
  onSelectionChange: (ids: string[]) => void;
}
```

**状态**：
- `loading`：加载状态，显示骨架屏
- `selectedRowKeys`：选中的行 keys
- `sortedInfo`：排序信息

#### 2.3.2 FeedbackDetail（反馈详情面板）

**职责**：侧边滑出展示单条反馈完整信息，支持状态编辑

**Props 接口**：
```typescript
interface FeedbackDetailProps {
  feedback: Feedback | null;
  open: boolean;
  onClose: () => void;
  onSave: (data: FeedbackUpdate) => Promise<void>;
}
```

**状态**：
- `editStatus`：是否编辑状态
- `handlerNotes`：处理备注
- `saving`：保存中状态

#### 2.3.3 StatCard（指标卡片）

**职责**：展示单个核心指标，支持点击跳转

**Props 接口**：
```typescript
interface StatCardProps {
  title: string;
  value: number | string;
  suffix?: string;
  trend?: {
    value: number;
    direction: 'up' | 'down';
  };
  icon: React.ReactNode;
  onClick?: () => void;
}
```

#### 2.3.4 TrendChart（趋势图表）

**职责**：展示反馈数量/评分的时间趋势

**Props 接口**：
```typescript
interface TrendChartProps {
  data: TrendData[];
  type: 'count' | 'rating';
  timeRange: '7d' | '30d' | '90d' | '1y';
  loading: boolean;

```

**ECharts 配置要点**：
- 使用 `tooltip` 组件展示悬停详情
- 使用 `grid` 组件控制图表边距
- 使用 `dataZoom` 组件支持时间范围选择

#### 2.3.5 FilterBar（筛选栏）

**职责**：多维度筛选条件管理

**Props 接口**：
```typescript
interface FilterBarProps {
  filters: FilterState;
  onFilterChange: (filters: FilterState) => void;
  routes: Route[];
  cities: string[];
  feedbackTypes: string[];
}
```

**状态管理**：
- 使用 Zustand 管理全局筛选状态
- 筛选变化时使用 300ms 防抖触发查询
- 支持筛选条件标签化展示和单独删除

### 2.6 API 请求封装

使用 Axios 封装请求实例，配置请求拦截器和响应拦截器：

```typescript
// client.ts
const client = axios.create({
  baseURL: import.meta.env.VITE_API_BASE_URL,
  timeout: 30000,
});

// 请求拦截器：内部工具，无需认证
client.interceptors.request.use((config) => {
  return config;
});

// 响应拦截器：统一错误处理
client.interceptors.response.use(
  (response) => response.data,
  (error) => {
    const message = error.response?.data?.message || '请求失败';
    message.error(message);
    return Promise.reject(error);
  }
);
```

#### 5.1.2 AI 客户端封装

```python
# utils/ai_client.py
import httpx
from typing import Optional, List, Dict, Any
from config import settings

class AIClient:
    """AI API 客户端，支持 Kimi 和 MiniMax"""

    def __init__(self, provider: str = "kimi"):
        self.provider = provider
        if provider == "kimi":
            self.api_key = settings.KIMI_API_KEY
            self.base_url = settings.KIMI_BASE_URL
            self.model = "moonshot-v1-8k"
        else:
            self.api_key = settings.MINIMAX_API_KEY
            self.base_url = settings.MINIMAX_BASE_URL
            self.model = "abab6-chat"

    async def chat(
        self,
        messages: List[Dict[str, str]],
        temperature: float = 0.7,
        max_tokens: int = 2000
    ) -> str:
        """发送对话请求"""
        async with httpx.AsyncClient(timeout=settings.AI_TIMEOUT) as client:
            response = await client.post(
                f"{self.base_url}/chat/completions",
                headers={
                    "Authorization": f"Bearer {self.api_key}",
                    "Content-Type": "application/json"
                },
                json={
                    "model": self.model,
                    "messages": messages,
                    "temperature": temperature,
                    "max_tokens": max_tokens
                }
            )
            response.raise_for_status()
            return response.json()["choices"][0]["message"]["content"]

# 单例模式
_kimi_client: Optional[AIClient] = None
_minimax_client: Optional[AIClient] = None

def get_kimi_client() -> AIClient:
    global _kimi_client
    if _kimi_client is None:
        _kimi_client = AIClient(provider="kimi")
    return _kimi_client

def get_minimax_client() -> AIClient:
    global _minimax_client
    if _minimax_client is None:
        _minimax_client = AIClient(provider="minimax")
    return _minimax_client
```

#### 5.2.3 分类服务实现

```python
# services/ai_service.py
import json
from typing import List, Dict, Any
from utils.ai_client import get_kimi_client

class AIService:
    """AI 分析服务"""

    async def classify_single(self, feedback_text: str) -> Dict[str, Any]:
        """单条反馈分类"""
        client = get_kimi_client()
        prompt = CLASSIFICATION_PROMPT.format(feedback_text=feedback_text)

        response = await client.chat(messages=[
            {"role": "user", "content": prompt}
        ])

        # 解析 JSON 响应
        result = json.loads(response)
        return result

    async def classify_batch(self, feedbacks: List[Dict]) -> List[Dict]:
        """批量反馈分类"""
        client = get_kimi_client()

        # 格式化反馈列表
        feedback_list = "\n".join([
            f"ID: {fb['id']}, 内容: {fb['feedback_text']}"
            for fb in feedbacks
        ])

        prompt = BATCH_CLASSIFICATION_PROMPT.format(feedback_list=feedback_list)

        response = await client.chat(messages=[
            {"role": "user", "content": prompt}
        ])

        # 解析 JSON 数组响应
        results = json.loads(response)
        return results
```

#### 6.1.2 宝塔站点配置

1. 创建前端站点：`feedback.xxx.com`
   - 填入域名
   - 选择纯静态
   - 根目录：`/www/wwwroot/robotaxi-feedback/client/dist`

2. 创建后端站点：`api.feedback.xxx.com`
   - 填入域名
   - 选择 Python 项目
   - 根目录：`/www/wwwroot/robotaxi-feedback/server`
   - 启动方式：Gunicorn/Uvicorn
   - 监听端口：8000

3. Nginx 反向代理配置

```nginx
# 前端 Nginx 配置
server {
    listen 80;
    server_name feedback.xxx.com;
    root /www/wwwroot/robotaxi-feedback/client/dist;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location /api {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}

# 后端 Nginx 配置
server {
    listen 80;
    server_name api.feedback.xxx.com;

    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
}
```
