# UI/UX 设计规范: Robotaxi 乘客反馈管理平台

## 1. 设计目标与原则

### 1.1 设计目标

本设计规范旨在为滴滴 Robotaxi 乘客反馈管理平台构建一套统一、高效、易用的界面体验。平台服务于滴滴自动驾驶运营团队，通过清晰的数据可视化与智能分析能力，帮助团队快速洞察乘客反馈、发现服务问题并提出优化建议。

核心目标：
- **效率优先**：减少用户操作步骤，快速获取关键信息
- **数据驱动**：通过可视化图表帮助用户理解数据趋势
- **智能辅助**：AI 功能帮助用户从海量反馈中提取有价值的信息
- **一致性**：保持全平台设计语言统一，降低学习成本

### 1.2 设计原则

| 原则 | 说明 | 应用场景 |
|------|------|----------|
| 清晰性 | 信息层次分明，重点突出 | 数据卡片突出核心指标，图表标注关键数据点 |
| 一致性 | 交互模式、视觉风格统一 | 全平台统一的组件样式和交互行为 |
| 可控性 | 用户对数据有掌控感 | 支持自定义筛选、排序、视图切换 |
| 效率性 | 减少认知负担，提升操作效率 | 快捷筛选、批量操作、键盘快捷键 |
| 可信性 | 数据准确可靠，呈现专业 | 明确的数据来源标注，图表刻度规范 |

## 2. 用户画像与使用场景

### 2.1 主要用户画像

#### 用户 A：运营数据分析员
- **背景**：负责日常反馈数据监控与分析
- **目标**：快速了解反馈概况，发现异常数据
- **行为特征**：
  - 每日多次查看反馈列表和仪表盘
  - 熟练使用筛选器定位特定类型反馈
  - 依赖数据图表进行趋势分析
- **痛点**：数据量大时难以快速定位问题，汇总报告耗时

#### 用户 B：运营管理专员
- **背景**：负责反馈处理闭环跟踪
- **目标**：确保每条反馈得到及时处理并反馈乘客
- **行为特征**：
  - 重点关注待处理反馈列表
  - 需要快速查看单条反馈详情
  - 与客服/运营团队协同处理问题
- **痛点**：处理状态不透明，跨部门协作效率低

#### 用户 C：产品与策略分析师
- **背景**：基于反馈数据提出产品优化建议
- **目标**：从反馈数据中挖掘产品改进机会
- **行为特征**：
  - 深度使用 AI 分析功能
  - 需要多维度交叉分析
  - 输出分析报告和优化建议
- **痛点**：手动分析效率低，难以发现隐藏规律

### 2.2 典型使用场景

| 场景 | 用户 | 目标 | 关键路径 |
|------|------|------|----------|
| 晨会数据概览 | 分析员 | 5分钟内了解昨日反馈概况 | 仪表盘 → 核心指标 → 趋势图表 |
| 投诉快速响应 | 专员 | 查看并处理紧急投诉 | 反馈列表 → 筛选待处理 → 详情 → 处理 |
| 周报数据分析 | 分析师 | 汇总本周反馈情况 | 仪表盘 → 多维图表 → AI 摘要生成 |
| 专项问题调研 | 分析师/专员 | 分析特定路线/时段的反馈 | 筛选器 → 多条件组合 → 列表详情 |

## 3. 信息架构

### 3.1 功能模块结构

```
乘客反馈管理平台
├── 模块零：工作台（Landing Page）
│   ├── 今日概览卡片
│   ├── 待处理提醒
│   ├── 趋势概览迷你图
│   ├── AI 摘要
│   └── 快捷入口
│
├── 模块一：反馈列表与管理
│   ├── 反馈列表（分页、排序）
│   ├── 多维度筛选面板
│   ├── 反馈详情抽屉/弹窗
│   └── 批量操作（导出、标记）
│
├── 模块二：数据统计仪表盘
│   ├── 核心指标卡片区
│   ├── 反馈趋势图表
│   ├── 分布分析图表
│   │   ├── 按路线分布
│   │   ├── 按时段分布
│   │   └── 按类型分布
│   └── 时间范围切换器
│
├── 模块三：AI 智能分析
│   ├── 自动分类面板
│   ├── 批量摘要生成
│   └── 产品建议生成
```


### 3.2 信息层级

| 层级 | 内容 | 优先级 | 更新频率 |
|------|------|--------|----------|
| L1 核心指标 | 总反馈数、平均评分、好评率、差评率 | 最高 | 实时 |
| L2 趋势数据 | 反馈数量趋势、评分趋势 | 高 | 每日更新 |
| L3 分布数据 | 路线分布、类型分布、时段分布 | 中 | 每日更新 |
| L4 明细数据 | 单条反馈详细内容 | 中 | 实时 |
| L5 AI 分析 | 分类标签、摘要、建议 | 低 | 按需生成 |

### 3.3 数据关联关系

```
乘客行程记录
    ├── route_id → 路线信息表
    ├── timestamp → 时间维度
    └── vehicle_id → 车辆信息表

乘客反馈记录
    ├── rating → 评分数据（1-5星）
    ├── feedback_type → 反馈类型枚举
    ├── category → AI 分类结果
    ├── content → 反馈文本内容
    └── sentiment → 情感分析结果
```
