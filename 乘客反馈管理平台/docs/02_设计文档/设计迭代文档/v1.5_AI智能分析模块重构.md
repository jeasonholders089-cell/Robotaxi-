## 1. 概述

### 1.1 设计背景

v1.5 版本对 AI 智能分析模块进行重构，实现完整的多步骤分析流水线，提供明确分析入口和实时进度反馈。

### 1.2 核心改动

| 改动点 | 当前状态 | 目标状态 |
|--------|----------|----------|
| 分析入口 | 自动加载数据 | 点击"开始分析"按钮触发 |
| 数据筛选 | 无直接关联 FilterBar | 完全基于 FilterBar 筛选条件 |
| 数据规模 | ≤100 条 | 默认 100 条，最多 2000 条 |
| 分析流程 | 单次调用 | 数据清洗 → 摘要 → 问题分析 → 建议 |
| 结果展示 | 两个独立模块 | 三块结果：摘要、问题分类、优化建议 |
| 用户反馈 | 无进度 | 实时进度条展示 |

### 1.3 数据筛选逻辑

数据来源**完全基于**顶部 FilterBar 的筛选条件：

| 筛选结果 | 分析数据量 |
|----------|-----------|
| 筛选结果 > 2000 条 | 取 2000 条进行分析 |
| 筛选结果 ≤ 2000 条 | 取全部筛选结果进行分析 |
| 无筛选条件（点击分析） | 默认取最新 100 条数据 |

## 4. 交互流程

### 4.1 完整分析流程

```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│   用户在顶部 FilterBar 设置筛选条件                          │
│         │                                                   │
│         ▼                                                   │
│   ┌─────────────────────────────────────────────────────┐  │
│   │  AI 智能分析  📊预估1,500条数据          [开始分析] │  │
│   └─────────────────────────────────────────────────────┘  │
│         │                                                   │
│         ▼ 点击"开始分析"                                    │
│   ┌─────────────┐                                           │
│   │ 分析中...   │  (按钮 loading 状态)                     │
│   └─────────────┘                                           │
│         │                                                   │
│         ▼ 分析完成（结果卡片直接显示）                       │
│   ┌─────────────────────────────────────────────────────┐  │
│   │  AI 摘要卡片                                        │  │
│   └─────────────────────────────────────────────────────┘  │
│         │                                                   │
│   ┌─────────────────────┐  ┌─────────────────────────┐      │
│   │  问题分类卡片         │  │  产品优化建议卡片       │      │
│   │  行驶体验   45%     │  │  [高] 优化变道策略...  │      │
│   │  车内环境   25%     │  │  [中] 增加充电接口...  │      │
│   └─────────────────────┘  └─────────────────────────┘      │
│                                                             │
│   分析完成后，按钮变为"重新分析"                             │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 4.2 按钮交互逻辑

| 点击次数 | 按钮文字 | 说明 |
|----------|----------|------|
| 首次点击 | "开始分析" | 触发第一次分析 |
| 后续点击 | "重新分析" | 重新发起分析 |

**状态流转**：

| 状态 | 触发条件 | 按钮文字 |
|------|----------|----------|
| idle | 初始状态 | "开始分析" |
| analyzing | 点击按钮 | "分析中..." + loading |
| completed | 分析成功完成 | "重新分析" |
| failed | 分析失败 | "重试" |
