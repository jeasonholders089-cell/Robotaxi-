## 1. 测试概述

本文档描述 v1.3 版本中 AI 分析功能优化的测试计划。

---

## 2. 测试范围

### 2.1 功能测试

| 模块 | 测试内容 |
|------|---------|
| AI 分析筛选联动 | "当前筛选"选项传递全局筛选参数 |
| AI 摘要接口 | rating、status、feedback_type、keyword 筛选 |
| AI 建议接口 | rating、status、feedback_type、keyword 筛选 |
| 问题分类饼图 | 使用 type_distribution 而非 suggestions |

### 2.2 回归测试

| 模块 | 测试内容 |
|------|---------|
| AI 分析"全部反馈"选项 | 不受筛选联动影响 |
| AI 分析"指定时间"选项 | 自定义时间范围正常工作 |
| 饼图渲染 | 数据显示正确 |

## 3. 测试用例

### 3.1 AI 分析筛选联动测试

| 用例ID | 描述 | 预期结果 |
|--------|------|---------|
| TC-AI-FILTER-001 | 选择"当前筛选"无筛选条件 | 分析全量数据 |
| TC-AI-FILTER-002 | 选择"当前筛选"城市=深圳 | 仅分析深圳数据 |
| TC-AI-FILTER-003 | 选择"当前筛选"评分≥4 | 仅分析高分数据 |
| TC-AI-FILTER-004 | 选择"当前筛选"状态=待处理 | 仅分析待处理数据 |
| TC-AI-FILTER-005 | 选择"当前筛选"反馈类型=行驶体验 | 仅分析该类型数据 |
| TC-AI-FILTER-006 | 选择"当前筛选"关键词=急刹车 | 仅分析包含关键词的数据 |
| TC-AI-FILTER-007 | 选择"当前筛选"组合筛选 | 按 AND 条件组合筛选 |

### 3.2 后端 API 测试

| 用例ID | 描述 | 预期结果 |
|--------|------|---------|
| TC-API-SUMMARY-001 | POST /ai/summary 带 rating_min/max | 返回符合评分范围的数据 |
| TC-API-SUMMARY-002 | POST /ai/summary 带 status 筛选 | 返回符合状态的数据 |
| TC-API-SUMMARY-003 | POST /ai/summary 带 feedback_type 筛选 | 返回符合类型的数据 |
| TC-API-SUMMARY-004 | POST /ai/summary 带 keyword 筛选 | 返回包含关键词的数据 |
| TC-API-SUGGEST-001 | POST /ai/suggestions 带 rating 筛选 | 建议基于筛选数据生成 |
| TC-API-SUGGEST-002 | POST /ai/suggestions 响应包含 type_distribution | 返回正确的分类分布 |

### 3.3 问题分类饼图测试

| 用例ID | 描述 | 预期结果 |
|--------|------|---------|
| TC-PIE-001 | 饼图数据来源验证 | 使用 type_distribution 数据 |
| TC-PIE-002 | 饼图数据显示 | 显示正确的问题类型和数量 |

### 3.4 回归测试

| 用例ID | 描述 | 预期结果 |
|--------|------|---------|
| TC-REG-AI-001 | "全部反馈"选项 | 分析全量数据，与 v1.2 行为一致 |
| TC-REG-AI-002 | "指定时间"选项 | 使用自定义时间范围 |
| TC-REG-AI-003 | 刷新按钮 | 重新生成分析 |
