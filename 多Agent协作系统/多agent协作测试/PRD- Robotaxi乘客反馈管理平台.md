## 1. 文档概述

### 1.1 文档目的

本文档旨在定义 Robotaxi 乘客反馈管理平台的功能需求、用户体验要求、数据规范及验收标准，为产品开发团队提供完整的需求依据。

### 1.2 适用范围

- 产品经理：需求确认与范围把控

- 开发团队：功能实现与接口定义

| 术语 | 定义 |
|------|------|
| Robotaxi | 自动驾驶出租车 |
| 反馈 | 乘客对单次行程体验的评价与文字描述 |
| 评分 | 乘客对行程的1-5星评分 |

## 2. 产品概述与背景

### 2.1 项目背景

随着滴滴自动驾驶 Robotaxi 服务在多个城市的开放运营，订单规模持续增长，乘客反馈数据快速积累。现有人工处理方式效率低下，难以从海量反馈中提取有价值的产品洞察。

团队需要一个统一的内部工具，实现反馈数据的集中管理、可视化分析以及AI驱动的智能分析能力，以数据驱动产品体验的持续优化。

### 2.2 产品定位

面向内部运营团队的**一站式反馈管理平台**，连接乘客体验数据与产品优化决策。

### 2.3 产品目标

| 目标层次 | 具体描述 |
|----------|----------|
| 核心目标 | 建立反馈数据的闭环管理：采集→分析→跟进→优化 |
| 功能目标 | 实现反馈的可视化分析、AI智能分类与产品建议生成 |
| 业务目标 | 提升运营团队处理效率，缩短问题发现到决策的周期 |

### 2.4 核心价值主张

1. **效率提升**：减少人工整理和分类的工作量
2. **洞察发现**：从数据中挖掘用户真实痛点
3. **决策支持**：为产品优化提供数据支撑

## 3. 用户场景与角色分析

### 3.1 目标用户

本平台服务于多角色混合使用场景，无需权限设计：

| 角色 | 核心诉求 |
|------|----------|
| 运营专员 | 高效查看今日反馈，处理投诉，标记跟进状态 |
| 产品经理 | 分析反馈趋势，发现体验问题，制定优化计划 |
| 数据分析师 | 提取数据，生成分析报告，追踪指标变化 |
| 客服主管 | 了解整体满意度，监控差评处理情况 |

### 3.2 核心使用场景

#### 场景一：日常反馈查看
- **触发**：运营专员每日上班
- **行为**：打开平台，查看今日新反馈列表
- **关注**：评分低于3分的差评、需要跟进的反馈
- **结果**：标记需要处理的反馈，开始跟进流程

#### 场景二：数据分析与汇报
- **触发**：周/月度复盘会议
- **行为**：打开仪表盘，查看关键指标和趋势图表
- **关注**：好评率变化、投诉热点分布、主要问题类型
- **结果**：导出数据或截图用于汇报

#### 场景三：问题根因分析
- **触发**：发现某指标异常（如某路线差评率上升）
- **行为**：通过筛选定位到相关反馈详情
- **关注**：具体乘客说了什么、问题的具体表现
- **结果**：形成问题假设，安排专项优化

#### 场景四：AI辅助产品优化
- **触发**：积累了一批反馈需要系统分析
- **行为**：选中反馈批次，调用AI分析能力
- **关注**：AI自动分类结果、智能摘要、产品建议
- **结果**：获取结构化分析结论，指导优化方向

### 3.3 用户痛点

| 痛点 | 当前状态（Before） | 期望状态（After） |
|------|-------------------|------------------|
| 反馈分散 | 反馈散落在各渠道，难以汇总 | 统一平台集中管理 |
| 分类耗时 | 人工阅读后打标签，效率低 | AI自动分类，即时完成 |
| 规律难发现 | 难以从个体反馈中发现共性 | 多维可视化分析揭示规律 |
| 跟进易遗漏 | 口头说需要跟进，后续忘记 | 状态标记+筛选不遗漏 |

| F1.1.2 | 分页控制 | 每页展示15条，提供页码导航和每页条数切换 | P0 |
| F1.1.3 | 时间排序 | 按行程时间倒序（最新在前）/正序排列 | P0 |
| F1.1.4 | 评分排序 | 按评分从高到低/从低到高排列 | P1 |
| F1.1.5 | 单条选择 | 点击行或复选框选中单条反馈 | P1 |
| F1.1.6 | 批量选择 | 支持全选当前页、全选搜索结果 | P1 |

**列表字段展示规范**：

| 字段 | 默认显示 | 宽度 | 对齐 |
|------|----------|------|------|
| 复选框 | 是 | 40px | 居中 |
| 反馈ID | 是 | 100px | 左对齐 |
| 评分 | 是 | 80px | 居中 |
| 评分标签 | 是 | 100px | 居中 |
| 反馈类型 | 是 | 120px | 居中 |
| 城市 | 是 | 100px | 居中 |
| 路线 | 是 | 200px | 左对齐 |
| 行程时间 | 是 | 160px | 居中 |
| 跟进状态 | 是 | 100px | 居中 |
| 操作 | 是 | 80px | 居中 |

**交互细节**：
- 评分显示为星级图标（★/☆）
- 评分低于3分整行高亮浅红色背景
- 鼠标悬停行显示"点击查看详情"提示
- 加载中显示骨架屏/loading动画

#### 4.1.2 多维度筛选

**功能描述**：提供多条件组合筛选，快速定位目标反馈。

**筛选维度**：

| 维度 | 筛选类型 | 说明 |
|------|----------|------|
| 评分范围 | 区间选择 | 支持1-5分区间筛选，如"3分以下"、"4-5分" |
| 时间范围 | 日期区间 | 选择开始和结束日期，默认展示近30天 |
| 城市 | 多选下拉 | 支持多选，默认全选 |
| 路线 | 搜索下拉 | 支持搜索和单选/多选 |
| 反馈类型 | 多选下拉 | 由AI分类自动生成 |
| 跟进状态 | 多选 | 待处理/处理中/已解决/已关闭 |
| 关键词 | 文本输入 | 模糊匹配反馈文本内容 |

**交互行为**：
- 筛选条件变化后自动触发搜索（防抖300ms）
- 展示当前筛选条件的标签，可单独删除
- 提供"重置全部"一键清除所有筛选
- 显示当前筛选条件下的结果数量

**边界条件**：
- 时间范围上限1年，超出提示
- 关键词最多50字符
- 至少保留一个有效筛选条件

#### 4.1.4 跟进状态管理

**功能描述**：对反馈进行状态标记和备注，记录处理过程。

**状态定义**：

| 状态 | 标识颜色 | 定义 | 状态流转 |
|------|----------|------|----------|
| 待处理 | 红色 | 新反馈，尚未开始处理 | →处理中/已关闭 |
| 处理中 | 橙色 | 正在处理中 | →已解决 |
| 已解决 | 绿色 | 问题已处理完毕 | →已关闭 |
| 已关闭 | 灰色 | 流程结束，无需再处理 | 终态 |

**状态流转规则**：
- 待处理 → 处理中：开始处理时更新
- 处理中 → 已解决：问题已处理完毕
- 已解决 → 已关闭：确认无需进一步跟进
- 已关闭 → 待处理：重新打开（特殊情况）

### 4.2 模块二：数据统计仪表盘

#### 4.2.1 核心指标卡片

**功能描述**：展示关键业务指标，一眼看全局。

**指标卡片清单**：

| 序号 | 指标名称 | 计算方式 | 展示形式 | 刷新 |

### 4.3 模块三：AI智能分析

#### 4.3.1 自动分类

**功能描述**：对单条或批量反馈文本进行AI自动分类。

**分类体系**：

| 一级分类 | 二级分类示例 | 说明 |
|----------|-------------|------|
| 行驶体验 | 急刹/顿挫、车道偏离、速度不稳、Autopilot退出 | 与驾驶行为相关 |
| 车内环境 | 温度不适、噪音大、异味、整洁度 | 车内物理环境 |
| 接驾体验 | 等待时间长、定位不准、司机响应 | 等待接驾阶段 |
| 路线规划 | 路线不合理、绕路、ETC收费、切换频繁 | 导航路线相关 |
| 安全感受 | 紧急刹车过于保守、风险预警不及时 | 安全感相关 |
| 费用问题 | 定价不合理、费用异常、优惠券无法使用 | 费用相关 |
| App问题 | 操作复杂、界面卡顿、功能异常 | 乘客端App |
| 客服体验 | 响应慢、态度差、解决问题能力弱 | 人工客服 |
| 其他 | 不适合以上分类的内容 | 兜底分类 |

**分类展示**：
- 每条反馈最多显示3个分类标签
- 分类标签可点击，触发对应筛选
- 分类结果保存到数据库，避免重复调用API

**批量分类**：
- 选中多条反馈后，点击"批量分类"按钮
- 显示分类进度条
- 完成后刷新列表显示新分类标签

#### 4.3.2 摘要生成

**功能描述**：对选定的批量反馈生成AI智能摘要。

**摘要类型**：

| 类型 | 适用场景 | 输出形式 |
|------|----------|----------|
| 单条摘要 | 单条反馈内容过长时 | 50字内核心观点 |
| 批量概览 | N条反馈的整体情况 | 100字内总结 |
| 问题聚焦 | 差评专项分析 | 50字内问题描述 |
| 用户之声 | 提炼原汁原味用户说词 | 20字内精选引用 |

**摘要生成配置**：

| 参数 | 可选值 | 说明 |
|------|--------|------|
| 摘要长度 | 短(50字)/中(100字)/长(200字) | 输出文字量 |
| 关注维度 | 全部/正面/负面/中性 | 按情感筛选后分析 |
| 时间范围 | 全部/近7天/近30天 | 按时间筛选 |
| 数量限制 | 最多100条/次 | 单次分析上限 |

**摘要展示**：
- 在仪表盘区域提供"AI摘要"卡片
- 手动刷新生成最新摘要
- 显示"生成时间"和"分析条数"
- 支持一键复制摘要文本

#### 4.3.3 产品建议

**功能描述**：基于反馈分析结果，自动生成产品优化建议。

**建议生成逻辑**：
1. 统计各分类的反馈数量和差评比例
2. 识别问题最严重的Top3分类
3. 结合问题严重程度生成优化建议

**建议输出格式**：

```
## 产品优化建议

### 优先级：高
**问题类型**：接驾体验-等待时长
**问题描述**：近30天相关反馈156条，占总反馈23%，其中差评率41%
**用户原声**：
  - "等了20分钟车才到，比预期晚太多了"
  - "定位不准，司机找不到我"
**优化建议**：
  1. 优化调度算法，减少乘客等待时间
  2. 升级定位精度，减少接驾偏差
  3. 增加预计到达时间的准确性展示

---
### 优先级：中
**问题类型**：车内环境-温度不适
...
```

**建议刷新**：
- 提供手动刷新按钮
- 每次刷新显示"分析时间"
- 最多保留最近3次分析记录
