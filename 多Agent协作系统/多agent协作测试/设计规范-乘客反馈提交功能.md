# 设计规范: 乘客反馈提交功能

## 1. 设计目标

为 Robotaxi 乘客提供简洁、直观的反馈提交体验，实现"一键反馈"的便捷操作。

### 核心目标
- **降低反馈门槛**：3步完成反馈提交，减少用户操作成本
- **信息自动补全**：行程信息自动关联，减少用户输入负担
- **即时反馈确认**：提交后立即显示工单号，增强用户信任感

### 设计原则
| 原则 | 说明 |
|------|------|
| 轻量化 | 最小化必填信息，仅收集核心数据 |
| 智能化 | 自动获取行程上下文，减少用户操作 |
| 即时性 | 每一步操作都有明确的状态反馈 |
| 一致性 | 与滴滴品牌视觉保持统一 |

## 2. 用户画像

### 主要用户群体

| 群体 | 特征 | 反馈场景 |
|------|------|----------|
| 高频乘客 | 经常使用 Robotaxi，对服务敏感 | 关注处理效率，希望快速提交 |
| 偶发乘客 | 偶尔使用，对流程不熟悉 | 关注操作引导，需要清晰指引 |
| 老年乘客 | 对 APP 操作不熟练 | 关注界面简洁，字号适中 |

### 用户痛点
- 不清楚如何反馈、反馈给谁
- 行程信息记不住，无法准确描述问题
- 反馈后不知道处理进度

## 3. 信息架构

### 页面结构

```
乘客反馈提交页面
├── Header（顶部导航）
│   ├── 返回按钮
│   ├── 页面标题
│   └── 行程关联提示

│   ├── 反馈类型选择
│   ├── 问题描述
│   ├── 图片上传
│   └── 行程信息展示

│   ├── 提交按钮
│   └── 隐私协议链接

```

### 数据流向

```
用户操作 → 表单验证 → API 提交 → 成功提示 → 跳转/停留
```

## 6. 组件设计

### 6.1 反馈类型选择器 (FeedbackTypeSelector)

**视觉规范**
| 属性 | 值 |
|------|-----|
| 圆角 | 8px |
| 高度 | 48px |
| 默认背景 | #F5F5F5 |
| 选中背景 | #FF6033 |
| 选中文字 | #FFFFFF |
| 默认文字 | #333333 |

**状态说明**
| 状态 | 视觉表现 |
|------|----------|
| 默认 | 灰色背景，深色文字 |
| 悬停 | 浅橙色背景 #FFF0EB |
| 选中 | 橙色背景，白色文字，图标高亮 |
| 禁用 | 50% 透明度 |

**交互行为**
- 点击切换选中状态
- 单选模式，只能选择一个类型
- 选中时有轻微缩放动画 (scale 0.98 → 1)

### 6.2 问题描述输入框 (DescriptionInput)

**视觉规范**
| 属性 | 值 |
|------|-----|
| 圆角 | 8px |
| 边框 | 1px solid #E0E0E0 |
| 聚焦边框 | 2px solid #FF6033 |
| 行数 | 5行 |
| 最大字数 | 500字 |

**状态说明**
| 状态 | 视觉表现 |
|------|----------|
| 默认 | 灰色边框 |
| 聚焦 | 橙色边框，轻微阴影 |
| 错误 | 红色边框，错误提示文字 |
| 禁用 | 灰色背景 |

**字数统计**
- 实时显示当前字数/最大字数
- 字数接近上限时（>450）显示警告色
- 到达上限时阻止继续输入

### 6.3 图片上传组件 (ImageUploader)

**视觉规范**
| 属性 | 值 |
|------|-----|
| 上传按钮尺寸 | 80px × 80px |
| 图片预览尺寸 | 80px × 80px |
| 圆角 | 8px |
| 边框 | 1px dashed #CCCCCC |
| 最多上传 | 3张 |
| 单文件大小 | ≤5MB |
| 支持格式 | JPG、PNG |

**状态说明**
| 状态 | 视觉表现 |
|------|----------|
| 空态 | + 图标居中，提示文字"添加图片" |
| 上传中 | 进度环动画 |
| 已上传 | 显示缩略图，右上角删除按钮 |
| 错误 | 红色边框，错误提示 |

**交互行为**
- 点击上传按钮触发文件选择
- 支持拖拽上传
- 点击已上传图片可预览（放大镜图标）
- 点击删除按钮移除图片（需二次确认）
- 上传失败时显示重试按钮

## 8. 状态说明

### 8.1 表单状态

| 状态 | 说明 | 视觉表现 |
|------|------|----------|
| 初始态 | 页面刚加载 | 占位符显示，引导文字 |
| 输入中 | 用户正在输入 | 聚焦态，实时字数统计 |
| 错误态 | 验证未通过 | 红色边框，错误提示 |
| 提交中 | 正在提交服务器 | 按钮 loading，禁止重复点击 |
| 成功态 | 提交完成 | 成功弹窗，工单号展示 |
| 网络错误 | 提交失败 | 错误弹窗，可重试 |

### 8.2 反馈类型选项状态

| 状态 | 视觉表现 |
|------|----------|
| 未选中 | 灰色背景 #F5F5F5，黑色图标+文字 |
| 悬停 | 浅橙色背景 #FFF0EB |
| 选中 | 橙色背景 #FF6033，白色图标+文字 |
| 禁用 | 50% 透明度，不可点击 |

### 8.3 图片上传状态

| 状态 | 视觉表现 |
|------|----------|
| 空态 | + 图标，灰色虚线边框 |
| 拖拽悬停 | 橙色虚线边框，浅橙色背景 |
| 上传中 | 圆形进度指示器 |
| 上传成功 | 缩略图预览，右上角删除按钮 |
| 上传失败 | 红色边框，重试按钮 |
| 达到上限 | 上传按钮隐藏 |

### 8.4 提交按钮状态

| 状态 | 背景色 | 可点击 |
|------|--------|--------|
| 未激活 | #CCCCCC | 否 |
| 可激活 | #FF6033 | 是 |
| 悬停 | #FF4500 | 是 |
| 点击中 | #CC4D29 + loading | 否 |
| 成功 | #52C41A | 否 |

### 8.5 异常状态处理

| 异常场景 | 处理方式 |
|----------|----------|
| 网络断开 | 弹窗提示"网络连接失败"，保留已填内容 |
| 提交超时 | 弹窗提示"提交超时，请重试" |
| 服务器错误 | 弹窗提示"服务器繁忙，请稍后重试" |
| 图片上传失败 | 显示失败图标，点击可重试 |
| 行程获取失败 | 显示"获取行程失败，请刷新重试" |
