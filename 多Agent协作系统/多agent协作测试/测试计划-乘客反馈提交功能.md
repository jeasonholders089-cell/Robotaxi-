# 测试计划: 乘客反馈提交功能

## 1. 测试范围

### 1.1 测试对象
乘客反馈提交功能模块（乘客端），包括以下功能点：

| 功能点 | 功能描述 | 优先级 |
|--------|----------|--------|
| 反馈类型选择 | 支持投诉/建议/表扬三种类型切换 | P0 |
| 问题描述 | 多行文本输入，最多500字 | P0 |
| 图片上传 | 支持上传最多3张图片 | P1 |
| 行程关联 | 自动关联当前/历史行程 | P0 |
| 提交确认 | 显示工单号等确认信息 | P0 |

### 1.2 测试边界

**范围内：**
- 反馈类型单选切换
- 问题描述文本输入与字数统计
- 图片上传、预览、删除
- 行程自动关联显示
- 提交成功后的工单号展示

**范围外：**
- AI自动分类功能（属于AI分析模块）
- 反馈列表展示与管理（属于运营端模块）
- 统计仪表盘功能

### 1.3 测试环境
- 客户端：Chrome最新版 / Edge最新版
- 屏幕分辨率：1366×768 及以上
- 网络：正常网络环境

## 2. 测试策略

### 2.1 测试类型覆盖

| 测试类型 | 覆盖内容 | 测试方法 |
|----------|----------|----------|
| 功能测试 | 各功能点的正常流程 | 黑盒测试 |
| 边界测试 | 字数限制、图片数量限制 | 边界值分析 |
| 异常测试 | 空提交、格式错误、网络异常 | 错误推测法 |
| UI测试 | 布局、样式、交互反馈 | 界面检查清单 |

### 2.2 测试数据准备

| 测试数据 | 用途 | 说明 |
|----------|------|------|
| 正常行程数据 | 基本提交流程 | 有效的行程ID、乘客信息 |
| 投诉类型文本 | 投诉提交 | 包含具体问题描述 |
| 建议类型文本 | 建议提交 | 包含改进建议 |
| 表扬类型文本 | 表扬提交 | 正面反馈内容 |
| 边界长度文本 | 字数边界测试 | 1字、500字、501字 |
| 图片文件 | 图片上传测试 | jpg/png格式, ≤5MB |

### 2.3 缺陷跟踪
- 使用缺陷等级定义（见本文档末尾）
- 发现缺陷立即记录并通知开发
- 修复后进行回归测试

## 缺陷等级定义

| 等级 | 定义 | 处理要求 |
|------|------|----------|
| P0 | 系统崩溃、数据丢失、无法提交反馈 | 立即修复 |
| P1 | 核心功能失效（类型切换失败、提交无工单号、行程关联失败） | 24小时内修复 |
| P2 | 次要功能缺陷（图片上传限制、文字截断提示、UI细节） | 72小时内修复 |
| P3 | 界面/体验问题（样式不一致、动画不流畅） | 排期修复 |
