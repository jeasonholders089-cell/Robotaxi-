# 演示任务：乘客反馈提交功能

## 任务背景

滴滴 Robotaxi 乘客反馈管理平台需要上线**反馈提交功能**，允许乘客在行程结束后提交反馈（投诉、建议、表扬）。

## 任务目标

完成从需求到上线的完整流程，验证多 Agent 协作机制有效性。

---

## 任务详情

### 功能需求

1. **反馈类型选择** - 乘客选择反馈类型（投诉/建议/表扬）
2. **问题描述** - 文字输入（最多 500 字）
3. **图片上传** - 最多 3 张图片
4. **行程关联** - 自动关联本次行程信息
5. **提交确认** - 显示提交成功，提供工单号

### 非功能需求

1. **响应时间** - 页面加载 < 2s，提交响应 < 1s
2. **兼容性** - iOS 14+、Android 8+、Chrome/Firefox/Safari
3. **可访问性** - 满足 WCAG 2.1 AA 标准

---

## 任务拆分

### 并行任务 A：产品设计
- **负责人**: 产品经理 Agent
- **输出**: PRD 文档、验收标准
- **耗时**: ~5min

### 并行任务 B：UI/UX 设计
- **负责人**: 设计师 Agent
- **输出**: 设计稿、组件规范、交互说明
- **依赖**: 需求分析完成
- **耗时**: ~5min

### 并行任务 C：技术实现
- **负责人**: 全栈工程师 Agent
- **输出**: 技术方案、代码实现
- **依赖**: 需求确认、设计稿完成
- **耗时**: ~10min

### 并行任务 D：测试验证
- **负责人**: 测试工程师 Agent
- **输出**: 测试计划、测试用例、测试报告
- **依赖**: 代码实现完成
- **耗时**: ~5min

---

1. ✅ PRD 文档
2. ✅ UI/UX 设计稿
3. ✅ 技术方案
4. ✅ 可运行代码

## 执行指南

### 启动多 Agent 协作

```bash
# 主控 Agent 执行以下命令启动并行协作
Skill("dispatching-parallel-agents")

# 然后分别启动各专业 Agent
Agent({ description: "产品经理 - 需求分析", prompt: "...", subagent_type: "general-purpose" })
Agent({ description: "设计师 - UI/UX设计", prompt: "...", subagent_type: "general-purpose" })
Agent({ description: "全栈工程师 - 技术实现", prompt: "...", subagent_type: "general-purpose" })
Agent({ description: "测试工程师 - 测试验证", prompt: "...", subagent_type: "general-purpose" })
```

### 协作时序

```
T+0min   主控 Agent 启动项目，发布任务
T+5min   产品经理 + 设计师 并行完成
T+10min  全栈工程师 完成开发
T+15min  测试工程师 完成测试
T+20min  主控 Agent 汇总交付
```

| 检查项 | 验收条件 |
